# FlowGuide - Техническое задание

## 1. Общее описание проекта

### 1.1 Название проекта
**FlowGuide** - Система записи и воспроизведения интерактивных пошаговых инструкций для Windows-приложений

### 1.2 Цель проекта
Создание Desktop-приложения для Windows, позволяющего:
- Записывать последовательность действий пользователя в любых Windows-приложениях
- Воспроизводить записанные инструкции в интерактивном режиме с визуальными подсказками
- Редактировать записанные инструкции

### 1.3 Целевая аудитория
- Специалисты по обучению и онбордингу сотрудников
- Технические писатели
- IT-поддержка
- Разработчики документации для корпоративного ПО

### 1.4 Примеры использования
- Обучение работе с 1С, SAP, внутренними корпоративными системами
- Создание интерактивных руководств для Office-приложений
- Документирование сложных бизнес-процессов
- Онбординг новых сотрудников

---

## 2. Архитектура системы

### 2.1 Компоненты системы

Система состоит из **трёх основных модулей**:

1. **FlowGuide.Core** - общая библиотека
2. **FlowGuide.Recorder** - приложение для записи
3. **FlowGuide.Player** - приложение для воспроизведения

### 2.2 Технологический стек

**Обязательные технологии:**
- **Платформа**: .NET 8.0
- **UI Framework**: WPF (Windows Presentation Foundation)
- **Target OS**: Windows 10/11 (x64)
- **Формат данных**: JSON для хранения гайдов
- **Автоматизация UI**: UI Automation API

**Дополнительные библиотеки:**
- `System.Drawing` - для работы с изображениями
- Win32 API - для глобальных хуков мыши/клавиатуры
- UI Automation - для поиска элементов интерфейса

## 3. Модуль FlowGuide.Core

### 3.1 Назначение
Общая библиотека, содержащая модели данных и сервисы, используемые как Recorder, так и Player.

### 3.2 Модели данных

#### 3.2.1 FlowGuide (Guide)
```csharp
class FlowGuide
{
    // Версионирование (v1.1+)
    int FormatVersion = 2         // Версия формата guide.json
    string MinPlayerVersion       // Минимальная версия Player для воспроизведения
    string CreatedWithVersion     // Версия Recorder, создавшего гайд
    
    // Основные поля
    string GuideId                // Уникальный ID гайда
    string Title                  // Название гайда
    string Description            // Описание
    string Author                 // Автор
    DateTime CreatedAt            // Дата создания
    List<Step> Steps              // Список шагов
}
```

> **Версионирование формата (v1.1+)**
> - `FormatVersion = 2` - текущая версия формата
> - При загрузке Player проверяет совместимость версий
> - Автоматическая миграция из формата v1 в v2


#### 3.2.2 Step
```csharp
class Step
{
    string StepId                        // Уникальный ID шага
    ActionType ActionType                // Тип действия
    TargetSelector TargetSelector        // Селектор элемента
    string FallbackImagePath             // Путь к скриншоту (для fallback поиска)
    string InstructionText               // Текст инструкции
    SystemActionType? SystemActionType   // Тип системного действия
    string MsSettingsCommand             // MS-Settings команда
    bool ShowAutoExecuteButton           // Показывать кнопку автовыполнения
}
```

#### 3.2.3 ActionType (enum)
- `LeftClick` - левый клик мыши
- `RightClick` - правый клик
- `DoubleClick` - двойной клик
- `TypeText` - ввод текста
- `KeyPress` - нажатие клавиши
- `SystemAction` - системное действие Windows

#### 3.2.4 SystemActionType (enum)
- `OpenStartMenu` - Win
- `OpenSettings` - Win+I
- `OpenCommandPrompt` - Win+R → cmd
- `OpenFileExplorer` - Win+E
- `OpenTaskManager` - Ctrl+Shift+Esc
- `OpenRun` - Win+R
- `OpenNotifications` - Win+N
- `OpenActionCenter` - Win+A

#### 3.2.5 TargetSelector
```csharp
class TargetSelector
{
    // Основные свойства UI Automation
    string AutomationId            // UI Automation ID элемента
    string Name                    // Имя элемента (может иметь несколько вариантов)
    string[] NameVariants          // Альтернативные имена (для разных локалей)
    string ClassName               // Класс элемента
    
    // Координаты
    Rect RecordedBoundingRect      // Физические пиксели, записанные на исходном ПК
    NormalizedRect NormalizedBoundingRect // Нормализованные координаты (0‑1) относительно виртуального экрана
    NormalizedRect NormalizedRectInWindow // Нормализованные координаты внутри окна (0‑1 относительно окна)
    int RecordedDPI                // DPI, при котором был сделан скриншот/запись
    Size RecordedScreenSize        // Размер экрана при записи (ширина, высота)
    
    // Иерархия
    List<AncestorInfo> Ancestors   // Цепочка родительских элементов
    
    // Дополнительные поля для будущего ранжированного поиска
    float? ScoreWeight             // Вес при ранжировании (может быть null)
}
```

> **Нормализованные координаты** позволяют корректно масштабировать шаги под любой DPI и разрешение.
> **NameVariants** решают проблему языковой независимости.
> **NormalizedRectInWindow** используется при поиске внутри окна, когда окно может перемещаться.


#### 3.2.6 AncestorInfo
```csharp
class AncestorInfo
{
    string AutomationId   // ID родителя
    string Name           // Имя родителя
    string ClassName      // Класс родителя
}
```

### 3.3 Сервисы

#### 3.3.1 UIAutomationService
**Назначение:** Поиск UI-элементов в запущенных приложениях

**Методы:**
- `GetElementAtPoint(Point screenPoint)` - найти элемент по координатам экрана
- `FindElementBySelector(TargetSelector selector)` - найти элемент по селектору
- `GetAncestors(AutomationElement element)` - получить цепочку родителей

**Алгоритм поиска:**
1. Попытка найти по AutomationId
2. Если не найден → поиск по Name + ClassName
3. Если не найден → поиск по цепочке предков (Ancestors)
4. Если не найден → fallback по координатам (RecordedBoundingRect)

#### 3.3.2 ScreenshotService
**Назначение:** Создание скриншотов элементов

**Методы:**
- `CaptureElement(Rect boundingRect, string outputPath)` - сделать скриншот области экрана
- `CaptureFullScreen(string outputPath)` - скриншот всего экрана

**Требования:**
- Формат: PNG
- Качество: без потери качества
- Скриншот должен быть сделан ПОСЛЕ завершения действия (с задержкой 100-200мс)

#### 3.3.3 StorageService
**Назначение:** Сохранение и загрузка гайдов

**Методы:**
- `SaveGuideAsync(FlowGuide guide, string directory)` - сохранить гайд
- `LoadGuideAsync(string directory)` - загрузить гайд

**Структура хранения:**
```
MyGuides/
├── FirstGuide/
│   ├── guide.json           # Метаданные и шаги
│   ├── step_1.png           # Скриншот шага 1
│   ├── step_2.png           # Скриншот шага 2
│   └── ...
└── SecondGuide/
    └── ...
```

**Формат guide.json:**
- Encoding: UTF-8
- Indent: 2 spaces
- PropertyNamingPolicy: CamelCase

#### 3.3.4 MouseHook
**Назначение:** Глобальный хук для отслеживания кликов мыши

**События:**
- `MouseClicked` - событие клика мыши

**Методы:**
- `Start()` - установить хук
- `Stop()` - снять хук

**Требования:**
- Использовать Win32 API `SetWindowsHookEx`
- Hook type: `WH_MOUSE_LL` (низкоуровневый хук)
- **КРИТИЧНО**: Обработчик хука должен быть максимально быстрым (<15ms)

## 4. Модуль FlowGuide.Recorder

### 4.1 Функциональные требования

#### FR-R-01: Главное окно
- Минималистичный интерфейс с кнопками:
  - "Записать новый гайд"
  - "Редактировать существующий гайд"
- Поля ввода:
  - Название гайда (обязательное)
  - Автор (обязательное, по умолчанию = %USERNAME%)
  - Описание (опциональное)
- Лог действий в нижней части окна

#### FR-R-02: Панель управления записью (Control Panel)
**Внешний вид:**
- Плавающее, всегда поверх всех окон (TopMost)
- Компактное (ширина ~200px)
- Полупрозрачный фон
- Перетаскиваемое мышью

**Кнопки:**
- "Start Recording" (зелёная)
- "Pause" (жёлтая)
- "Stop" (красная)

**Состояния:**
- Готово к записи: только Start активна
- Идёт запись: Start недоступна, Pause и Stop активны
- Пауза: все кнопки активны

#### FR-R-03: Визуальная подсветка (Recording Overlay)
**Требования:**
- Полноэкранное прозрачное окно
- ClickThrough (пропускает клики мыши)
- Красная рамка вокруг элемента под курсором
- Толщина рамки: 3px
- Цвет: #FF0000
- Обновление: в реальном времени (60 FPS желательно)

**Калибровка:**
- Padding: 8px (настраиваемое)
- Manual offset X: 10px (настраиваемое)
- Manual offset Y: 10px (настраиваемое)

#### FR-R-04: Запись действий
**Записываемые действия:**
1. Left Click - основное действие
2. Системные действия (Start Menu, Settings и т.д.)

**Процесс записи клика:**
1. Пользователь кликает на элемент
2. Система:
   - Определяет элемент под курсором (UIAutomationService)
   - Создаёт TargetSelector с AutomationId, Name, ClassName, Ancestors
   - Делает скриншот элемента (после задержки 150ms)
   - Сохраняет BoundingRect
   - Автоматически генерирует InstructionText
3. Шаг добавляется в список

**Автогенерация InstructionText:**
```
Шаблоны:
- Для кнопок: "Нажмите кнопку '{Name}'"
- Для текстовых полей: "Введите текст в поле '{Name}'"
- Для чекбоксов: "Установите флажок '{Name}'"
- По умолчанию: "Кликните на '{Name}'"
```

#### FR-R-05: Определение системных действий
**Автоматическое распознавание:**
- Клик на кнопку "Пуск" → SystemAction: OpenStartMenu
- Клик на иконку "Параметры" → SystemAction: OpenSettings
- И т.д.

**Проверка:**
```csharp
if (element.AutomationId == "StartButton")
    → SystemActionType.OpenStartMenu
if (element.Name.Contains("Параметры") && element.ClassName == "...")
    → SystemActionType.OpenSettings
```

#### FR-R-06: Редактор шагов (Step Editor Window)
**Функциональность:**
- Список всех записанных шагов
- Drag-and-drop сортировка
- Кнопки для каждого шага:
  - "Edit" - редактировать шаг
  - "Delete" - удалить шаг
  - "Move Up/Down" - изменить порядок

**Отображение шага:**
```
[1] Нажмите кнопку "ОК"
Type: LeftClick
```

#### FR-R-07: Редактор отдельного шага (Step Edit Dialog)
**Поля:**
- Instruction Text (TextBox, многострочный)
- Action Type:
  - Radio: Normal Click
  - Radio: System Action
- System Action Settings (видимо только для System Action):
  - ComboBox: Тип системного действия
  - TextBox: MS-Settings Command (optional)
  - CheckBox: Show 'Auto Execute' button

**Кнопки:**
- Save (зелёная)
- Cancel (серая)

#### FR-R-08: Сохранение гайда
**Требования:**
- Гайд сохраняется в `%USERPROFILE%\Documents\FlowGuide\{GuideName}\`
- Автоматическое создание папки
- Проверка на дубликаты имён
- Сохранение всех скриншотов
- Уведомление об успешном сохранении

## 5. Модуль FlowGuide.Player

### 5.1 Функциональные требования

#### FR-P-01: Главное окно
**Интерфейс:**
- Кнопка "Load Guide" - выбрать guide.json
- Список доступных гайдов (optional)
- Информация о выбранном гайде:
  - Название
  - Автор
  - Количество шагов
- Кнопка "Start Playback"

#### FR-P-02: Оверлей воспроизведения (Playback Overlay)
**Характеристики:**
- Fullscreen, прозрачное окно
- TopMost (поверх всех окон)
- Click-through с защитой от ошибочных кликов

**Визуальные элементы:**
1. **Затемнение (Spotlight effect)**
   - Полупрозрачный чёрный фон (#80000000)
   - "Дырка" вокруг целевого элемента
   - Using: CombinedGeometry с Exclude операцией

2. **Рамка подсветки**
   - Цвет: #00FF00 (зелёный)
   - Толщина: 4px
   - Анимация: мигание (optional)
   - Отступ: 8px от элемента
   - Калибровка: manualOffsetX=10, manualOffsetY=10

3. **Окно инструкции (Instruction Bubble)**
   - Позиционирование: под элементом (или над, если не влезает)
   - Фон: белый с тенью
   - Текст: крупный, читаемый
   - Макс. ширина: 400px
   - **Функция перетаскивания**: пользователь может перетащить окно мышью

#### FR-P-03: Позиционирование Instruction Bubble
**Алгоритм:**
1. По умолчанию: под целевым элементом
2. Если не влезает снизу → над элементом
3. Если не влезает сверху → у верхнего края экрана
4. Горизонтальная проверка: отступ 10px от краёв экрана
5. Использовать `ActualWidth/ActualHeight` для точного расчёта

**Перетаскивание:**
- Курсор: SizeAll (четырёхсторонняя стрелка)
- Drag: LeftMouseButton Down → Move → Up
- Ограничение: окно должно оставаться в пределах экрана (margin 10px)

#### FR-P-04: Защита от ошибочных кликов (Click Guard)
**Принцип:**
- По умолчанию overlay **блокирует** все клики
- Пользователь может кликнуть **только** на целевой элемент
- Клики вне целевого элемента игнорируются

**Реализация:**
- Обработчик `MouseLeftButtonDown` на overlay
- Проверка: `HitTest` → если клик внутри целевой области → `e.Handled = false`
- Иначе → `e.Handled = true` (клик заблокирован)

#### FR-P-05: Поиск и выделение элемента
**Алгоритм:**
1. Использовать `UIAutomationService.FindElementBySelector()`
2. Если не найден → использовать fallback по координатам
3. Если элемент движется → background thread отслеживает изменения позиции
4. Обновлять рамку подсветки в реальном времени

**Background Search:**
```csharp
Task.Run(async () => {
    while (isPlaying) {
        var element = FindElement();
        if (element != null) {
            Dispatcher.Invoke(() => UpdateSpotlight(element));
        }
        await Task.Delay(100); // 10 FPS
    }
});
```

#### FR-P-06: Переход между шагами
**Навигация:**
- Автоматически: после успешного клика → следующий шаг
- Вручную: кнопки "Next", "Previous" (опционально)
- Hotkey: Esc → закрыть Player

**Индикация прогресса:**
```
Шаг 3 из 15
```

#### FR-P-07: Обработка системных действий
**Для SystemAction:**
- Отображать InstructionText с объяснением
- Показывать кнопку "Auto Execute" (если ShowAutoExecuteButton = true)
- При клике на кнопку → автоматически выполнить действие:
  ```csharp
  switch (SystemActionType) {
      case OpenStartMenu:
          SendKeys.SendWait("{LWIN}");
          break;
      case OpenSettings:
          if (!string.IsNullOrEmpty(MsSettingsCommand))
              Process.Start(MsSettingsCommand);
          else
              SendKeys.SendWait("{LWIN}i");
          break;
      // ...
  }
  ```

#### FR-P-08: Переключение оверлея
**Требования:**
- CheckBox в углу экрана: "Show Overlay"
- Hotkey: H → toggle overlay visibility
- При скрытии оверлея:
  - Затемнение исчезает
  - Рамка подсветки остаётся
  - Instruction Bubble остаётся

#### FR-P-09: Кнопка закрытия
- Красная кнопка "✕" в правом верхнем углу
- Hotkey: Esc
- Confirm dialog: "Вы уверены, что хотите прервать воспроизведение?"

## 7. Нефункциональные требования

### 7.1 Производительность

**Recorder:**
- Задержка отклика рамки подсветки: < 50ms
- Потребление CPU в idle: < 5%
- Потребление RAM: < 100 MB

**Player:**
- Поиск элемента: < 500ms
- Обновление рамки подсветки: 60 FPS (желательно)
- Потребление RAM: < 150 MB

### 7.2 Надёжность
- Обработка ошибок:
  - Element not found → fallback на координаты
  - Screenshot failed → продолжить без скриншота
  - Save failed → показать сообщение об ошибке
- Автосохранение (опционально)

### 7.3 Безопасность
- **КРИТИЧНО**: Приложения требуют прав администратора для:
  - Установки глобальных хуков (MouseHook)
  - Доступа к UI Automation в защищённых приложениях
- app.manifest: `<requestedExecutionLevel level="requireAdministrator" />`

### 7.4 Совместимость
**Поддерживаемые приложения:**
- Стандартные Windows-приложения (Explorer, Settings)
- Office (Word, Excel, Outlook)
- 1С
- Браузеры (ограниченно)
- WPF приложения
- WinForms приложения

**НЕ поддерживаются:**
- Web-приложения внутри браузера (DOM не доступен)
- Игры в полноэкранном режиме
- Приложения с нестандартным UI (custom rendering)

### 7.5 Локализация
- Базовый язык: Русский
- Поддержка английского: опционально
- Все UI-тексты вынесены в константы

## 10. Риски и ограничения

### 10.1 Технические риски
**Риск 1:** UI Automation API не работает с некоторыми приложениями
- **Решение**: Fallback на координаты + скриншоты

**Риск 2:** Глобальные хуки требуют прав администратора
- **Решение**: Явно указать в документации

**Риск 3:** Окна могут двигаться между шагами
- **Решение**: Background thread для отслеживания позиции

**Риск 4:** DPI Scaling может влиять на точность координат
- **Решение**: Использовать физические пиксели, учитывать DPI

### 10.2 Ограничения
- Не работает с web-приложениями
- Не работает с приложениями без UI Automation
- Требует прав администратора
- Только Windows 10/11
